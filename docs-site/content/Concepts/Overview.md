Symphony operates within an orchestration layer, strategically positioned atop preexisting tools and services. To establish uniformity amidst the diverse edge environment, Symphony relies on a set of fundamental high-level abstractions. These encompass state-seeking, graph, workflow, and orchestration models. These foundational abstractions empower Symphony to deliver robust functionalities across various technologies and platforms while maintaining a concise object model.

## Abstractions

* [Information graph](../abstractions/info_graph/): How Symphony organizes, accesses, and synchronizes enterprise information.
* [Orchestration model](../abstractions/orch_model/): How Symphony defines the components that make up a scenario.
* [State seeking](../abstractions/state_seeking/): How Symphony brings the current state of a system to the desired state.
* [Workflows](../abstractions/workflows/): How Symphony manages multi-stage deployment scenarios.
